调试说明：

1) 打开站点
   http://localhost:8000

2) 打开开发者工具（Chrome/Edge：F12）-> Console / Elements

3) 在 Console 中观察以下关键日志（已添加的 debug 输出）
   - [spa-router] dispatching spa:page:loaded
   - [spa-router] calling loadReadme()/initLightbox()/siteLightbox.refresh()（如果出现）
   - [readme] spa:page:loaded received
   - [readme] loadReadme called, protocol=file: 或 http:
   - [readme] container click
   - [lightbox] initLightbox: scanning candidates
   - [lightbox] refreshAll called
   - [lightbox] candidates before refresh: N

4) 验证步骤
   - 点击侧边栏的 README，观察 Console 是否出现上述日志；如果没有，回到这里并把 Console 中的全部日志复制给我。
   - 在 README 页面里点击图片，观察 Console 是否输出 [readme] container click，然后检查 lightbox 是否打开。

5) 若 lightbox 未打开或 filmstrip 为空：
   - 在 Console 执行：
     document.querySelectorAll('main img, #aside-stack img').length
     // 这应返回候选图片数量
   - 执行：document.getElementById('image-lightbox') 并把结果粘贴回来（会显示元素结构）

6) 把 Console 中与上述关键日志对应的所有行复制给我，另外粘贴上述两条命令的返回值。

备注：我已经在代码中做了自动化尝试（spa-router 在发事件后会尝试调用 `loadReadme()` / `initLightbox()` / `siteLightbox.refresh()`），但我们仍需要浏览器的运行时日志来定位剩余问题。
